Projects

I am working on a project that utilizes the publicly available and popular SDL2 library to create a 3-D renderer. The project is still in progress, but I am working right now on programming the ability to shade 3-D projected objects to the screen. I would like to bring the project to a point in which I can simulate basic physics interactions between basic objects on the screen.

With this being said, when it comes to my interest regarding projects, most of my interest lies in low level implementations. The challenge of using only SDL2 and as such forgoing a graphical API such as OpenGL provides me with additional challenges.

One of the main challenges lies in the fact that SDL2 is designed to be paired with a graphical API. It does not provide a 3-D coordinate space, nor does it provide any default implementations of some of the more important functions of a traditional 3-D renderer.

With this in mind, the challenge lies in coding low level implementations of needed functions to make a 3-D image on a 2-D screen possible. This involves the creation of a projection matrix, definitions for mesh objects, linear transformation matrices, code involved in shading and lighting calculations, and even the implementation of a 3-D world space as SDL2 only provides a 2-D coordinate space (which is defined by the resolution of the window SDL2 creates).

The creation of a project of this scale has been hard, and it has taken me many hours to even get the program to the point at which is being displayed now, but it has taught me valuable lessons through the many mistakes I've made.

One such mistake I made early on was making the mistake of not making my code more modular. I put many functionalites together that did not belong together, and this caused me many issues down the line. The process of refactoring my earlier written code was quite frankly a nightmare, but as I stated before, it taught me a valuable lesson: utilize modular programming techniques as early on as you can when you work on a project of this size.

I have become well-aqainted with GitHub through the process of crerating this project. Being specific and using correct terminology when making commits has become a prime concern of mine as I have progressed further and further thorugh this project.

I want to demonstrate that I am capable of correctly utilizing GitHub, because that is a valuable skill in this profession, but also it provides me with another valuable organizational tool: Version control.
